- 我的默认语言是中文（除非有特别要求），行事风格是简洁、实用、不废话。

## 我如何在这个仓库中工作

### 仓库结构约定

```
AGENTS.md         # 本文件：身份、规则、工作方式（核心）
MEMORY.md         # 长期记忆：持久偏好、重要决策、背景知识
.agents/skills/   # 项目级技能目录（统一入口）
memory/           # 结构化记忆目录
  daily/          # 按日期的临时记录（YYYY-MM-DD.md）
  tasks/          # 进行中或已归档的任务记录
  context/        # 特定主题的背景知识（项目、技术、偏好等）
README.md         # 仓库对外说明
```

### 工作原则

1. **文件即记忆**：重要信息不只留在对话里，必须写入对应文件。
2. **MEMORY.md 是长期记忆**：只记录持久有价值的内容，不堆砌临时细节。
3. **memory/daily/ 是短期日志**：每日工作记录，定期归纳后可删除或存档。
4. **最小侵入**：不为了"有记录感"而创建无用文件；只在真正需要时才写。
5. **技能统一治理**：项目级技能统一放在 `.agents/skills/`，禁止散落在其他目录。

### 技能发现、安装与使用机制

1. **技能目录规范**
   - 根目录固定：`.agents/skills/`
   - 每个技能必须使用独立目录：`.agents/skills/<skill-name>/`
   - 每个技能目录至少包含 `SKILL.md`（描述用途、输入输出、使用边界）
   - 如需依赖安装，使用 `install.sh`（可选）

2. **发现机制（Discover）**
   - 主代理在开始任务前，先扫描 `.agents/skills/` 下的一层子目录作为可用技能集
   - 只把包含 `SKILL.md` 的目录视为有效技能
   - 技能总览由 `.agents/skills/README.md` 维护

3. **安装机制（Install）**
   - 仓库通过 `.github/workflows/copilot-setup-steps.yml` 自动发现技能并执行各技能的 `install.sh`
   - `install.sh` 必须可重复执行（幂等），失败即返回非零退出码
   - 禁止在安装脚本中写入敏感信息或修改仓库外的关键环境

4. **使用机制（Use）**
   - 主代理按任务匹配技能，优先复用已有技能而不是重复造轮子
   - 使用技能前先阅读对应 `SKILL.md`，严格遵守其边界与约束
   - 如技能不适配当前任务，明确回退到通用流程，不强行调用

## 我如何管理任务与记忆

### 任务管理

- 接到任务时，先在 `memory/tasks/` 下创建任务文件（如 `memory/tasks/TASK-001.md`）。
- 文件包含：任务描述、当前状态（进行中 / 完成 / 搁置）、关键决策、产出链接。
- 任务完成后，在文件中标记状态并记录结论。

### 记忆分层

| 层级 | 文件位置 | 内容 | 更新频率 |
|------|----------|------|----------|
| 长期记忆 | `MEMORY.md` | 偏好、约定、重要结论 | 按需，经过筛选 |
| 主题记忆 | `memory/context/*.md` | 特定项目/技术的背景 | 按需 |
| 日常记录 | `memory/daily/YYYY-MM-DD.md` | 当日工作流水 | 每次对话 |
| 任务记录 | `memory/tasks/*.md` | 任务状态与进展 | 任务生命周期内 |

---

## 每次任务完成后的收尾动作

完成任务后，我需要：

1. **更新任务文件**：把结论、产出、未尽事项写进 `memory/tasks/` 对应文件。
2. **提炼长期价值**：如有新的偏好、约定或重要决策，更新 `MEMORY.md`。
3. **记录日志**：在 `memory/daily/YYYY-MM-DD.md` 追加本次工作摘要（一两句话即可）。
4. **提交文件**：确保所有记忆文件已提交，不把信息只留在当前对话。
